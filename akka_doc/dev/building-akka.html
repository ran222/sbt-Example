编译 Akka

本页介绍如何从最新的源码编译和运行Akka.

获取源码

Akka 使用 Git 并托管在 Github上.

首先你必须在你的机器上安装有Git. 从 http://github.com/akka/akka克隆源码.

例如:

git clone git://github.com/akka/akka.git

如果以前已经克隆过源码库，只需要做 git pull:

git pull origin master

sbt - Simple Build Tool

Akka 使用优秀的 sbt 工具. 首先下载和安装sbt. 参考 sbt setup 文档.

用来编译Akka的sbt指令都在下面. 要更多地了解 sbt并将它用于自己的项目一定要阅读 sbt documentation.

Akka sbt 编译文件位于 project/AkkaBuild.scala.

编译 Akka

首先确定你当前在akka代码目录下:

cd akka

编译

要编译所有Akka核心模块使用 compile 指令:

sbt compile

用test 指令运行所有的测试:

sbt test

如果编译和测试都成功了，你已经拥有了可以运行的最新Akka开发版本.

并行执行

缺省状况下测试是顺序执行的。也可以并行执行以缩短编译时间, 前提是硬件能够提供足够的内存和CPU。将下面的系统属性加入sbt的启动脚本来激活并行执行:

-Dakka.parallelExecution=true

发布到本地Ivy库

如果你希望将编译出来的包部署到本地Ivy库（例如，从另一个sbt项目中使用它），使用 publish-local 指令:

sbt publish-local

sbt 交互式模式

注意在上面的例子中我们使用 sbt compile 和 sbt test 这些指令, 但sbt还有一种交互式模式。如果你只运行 sbt 就将进入交互式sbt提示，可以在里面直接输入指令。 这节省了每条指令都启动新的JVM实例的时间，会更加快速和方便。

例如，上例中编译AkkaFor的过程更多人是这么做的:

% sbt
[info] Set current project to default (in build file:/.../akka/project/plugins/)
[info] Set current project to akka (in build file:/.../akka/)
> compile
...
> test
...

sbt 批处理模式

还可以将多条指令组合到一次请求中. 例如，测试并将Akka发布到本地Ivy库中，可以这么做:

sbt test publish-local

依赖

你可以看看的 sbt update 时Ivy解决依赖的信息， 并在 ~/.ivy2/cache 找到相关的文件. 例如， ~/.ivy2/cache/com.typesafe.akka-akka-remote-compile.xml 文件包含了 在编译akka-remote模块时解决依赖的信息。如果你在浏览器中打开这个文件你会看到一个依赖的可导航视图.
